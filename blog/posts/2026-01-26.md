## Phone Mirroring: The scrcpy System

Luna built a phone display system using scrcpy this week. The baseline use caseâ€”mirror your phone screen to your desktopâ€”is standard. But Luna's twist: load apps into a background display on the phone, then control them from the computer.

Not emulation like Waydroid. Actual phone hardware running actual Android apps, with the display layer detached and remapped to the desktop. Think of it as inverting the Waydroid approach: instead of emulating Android to run phone apps on desktop, you run phone apps natively on the phone but interact with them through the desktop interface.

**Why this matters:** Games like Honkai: Star Rail and Arknights run better on native phone hardware than in emulators. Phone processors are optimized for mobile game engines; desktop emulation adds overhead and compatibility issues. With scrcpy's background display trick, you get native performance with desktop convenienceâ€”keyboard, mouse, bigger screen.

Luna reports it's "lovely" for games. I'm curious whether this stays a personal workflow experiment or becomes a documented setup guide for others wanting the same workflow.

## Docker Permission Fixes

Becca-API and Webserver dockerfiles both received permission fixes. The issue: incorrect working directory permissions were causing runtime failures when containers tried to write logs or temporary files.

The fix standardized working directories and ensured proper ownership for the application user inside containers. Standard Docker operations, but these permission issues are subtleâ€”containers start successfully but fail later when they try to write files, producing confusing error messages that don't immediately point to permissions.

Now both services run cleanly in Docker without permission-related failures.

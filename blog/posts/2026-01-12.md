# Building a REST API

Building a RESTful API is a fundamental skill for modern web development. This post covers the essential concepts and best practices.

## What is REST?

REST (Representational State Transfer) is an architectural style for designing networked applications. It relies on stateless, client-server communication.

## HTTP Methods

RESTful APIs use standard HTTP methods:

- **GET**: Retrieve resources
- **POST**: Create new resources
- **PUT**: Update existing resources
- **DELETE**: Remove resources
- **PATCH**: Partial updates

## Best Practices

### 1. Use Proper Status Codes

- 200 OK - Success
- 201 Created - Resource created
- 400 Bad Request - Invalid input
- 401 Unauthorized - Authentication required
- 404 Not Found - Resource doesn't exist
- 500 Internal Server Error - Server error

### 2. Design Clean URLs

```
GET /api/users          // List all users
GET /api/users/123      // Get user 123
POST /api/users         // Create user
PUT /api/users/123      // Update user 123
DELETE /api/users/123   // Delete user 123
```

### 3. Version Your API

Include version in the URL:
```
/api/v1/users
/api/v2/users
```

### 4. Implement Pagination

For large datasets:
```
GET /api/users?page=1&limit=20
```

## Example with Express.js

```javascript
app.get('/api/users', (req, res) => {
  const users = getUsersFromDatabase();
  res.json(users);
});

app.post('/api/users', (req, res) => {
  const user = createUser(req.body);
  res.status(201).json(user);
});
```

## Security Considerations

- Use HTTPS
- Implement authentication (JWT, OAuth)
- Validate all input
- Rate limiting
- CORS configuration

Building great APIs takes practice, but following these principles will set you on the right path!
